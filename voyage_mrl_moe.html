Matryoshka Representation Learning
All voyage-4 models are trained with MRL: the first d dimensions of any embedding are already a valid d-dimensional embedding. One embed call — store once at full 2048 dims — then truncate at serving time with zero re-indexing cost. Three independent levers control quality, storage, and query cost.
Lever 1 — Shared Embedding Space
Which model embeds your queries?
Documents indexed with voyage-4-large at full accuracy. All voyage-4 models share one embedding space — any model can serve queries against the same index. Asymmetric indexing: pay large-model cost once at write time, lite-model cost on every query.
Lever 2 — MRL Truncation
How many dimensions do you keep?
Store at 2048 dims once; truncate at query time or set output_dimension=N at embed time. Lower dims → smaller ANN index, faster search, lower memory footprint. Scores below are Recall@10 relative to the 2048-dim float32 baseline on MTEB/BEIR.
Lever 3 — Quantization
What numeric precision for storage?
voyage-4 models are quantization-aware trained — int8 loses <1% NDCG vs float32. Binary (1 bit/dim) cuts storage 32× vs float32 but costs ~3–5% NDCG; pair with a re-rank pass for production use.

Mixture of Experts — voyage-4 Architecture
voyage-4 uses a sparse MoE feed-forward layer: 8 specialist experts, 2 active per token selected by a learned gating network. Only 25% of parameters execute per forward pass — the model achieves dense-quality output at sparse-model compute cost. Expert specialisation also improves domain coverage vs a single dense FFN of equivalent size.

MoE × MRL — how they compose
MoE and MRL are orthogonal optimisations — they stack independently.
AxisWhat it controlsTrade-off knob
MoE routing Compute cost & domain quality at model inference time Number of active experts (fixed at 2 in voyage-4)
MRL truncation Storage cost & ANN search speed at index / retrieval time Output dimension: 64 → 2048
Quantization Vector storage footprint fp32 → int8 → binary
Choosing 512-dim int8 MRL doesn't change which MoE experts fire — MoE happens inside the encoder; MRL happens to the output vector. They are independently tunable.
Deployment Decision Guide
Three independent axes — model tier, embedding dimension, quantization — each chosen against a different constraint. Mix and match freely; the shared embedding space means you can change any axis without re-indexing the corpus.
